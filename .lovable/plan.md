

# План: Telegram-уведомления о подписках

## Обзор
Реализация системы уведомлений в Telegram бот `@subday_lgbot` для двух сценариев:
1. **При активации подписки** — уведомление "Подписка активирована. X кофе на Y дней"
2. **При низком остатке** — уведомления при остатке <7, <5, <3 кофе: "У вас по подписке осталось X кофе на Y дней"

## Архитектура решения

```text
┌─────────────────────────────────────────────────────────────────┐
│                    Точки активации уведомлений                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. Активация подписки:                                        │
│      ├── Админ панель (AdminUsersPage.tsx)                      │
│      └── Будущая оплата (интеграция с платёжной системой)       │
│                                                                 │
│   2. Уведомление об остатке:                                    │
│      └── Partner QR Scan (partner-scan-qr Edge Function)        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│              send-subscription-notification                      │
│                   (новая Edge Function)                         │
├─────────────────────────────────────────────────────────────────┤
│  Входные данные:                                                │
│  - userId: UUID пользователя                                    │
│  - type: 'activated' | 'low_balance'                            │
│  - cupsCount: количество кофе                                   │
│  - daysCount: дней до истечения                                 │
│  - remaining?: остаток (для low_balance)                        │
│                                                                 │
│  Логика:                                                        │
│  1. Получить данные пользователя (имя, телефон)                 │
│  2. Сформировать сообщение                                      │
│  3. Отправить в Telegram используя NOTIFICATION_BOT_TOKEN       │
│     и NOTIFICATION_CHAT_ID                                      │
└─────────────────────────────────────────────────────────────────┘
```

## Детальный план реализации

### Шаг 1: Создание Edge Function `send-subscription-notification`

Создать новую Edge Function для централизованной отправки уведомлений о подписках:

**Файл:** `supabase/functions/send-subscription-notification/index.ts`

**Функционал:**
- Принимает тип уведомления (`activated` или `low_balance`)
- Получает данные пользователя из базы
- Формирует соответствующее сообщение
- Отправляет в существующий Telegram канал через `NOTIFICATION_BOT_TOKEN`

**Формат сообщений:**
- Активация: `☕ Подписка активирована\n👤 {name} ({phone})\n📦 {cups} кофе на {days} дней`
- Низкий остаток: `⚠️ Низкий остаток\n👤 {name} ({phone})\n📦 Осталось {remaining} кофе на {days} дней`

### Шаг 2: Обновление Admin Panel (AdminUsersPage.tsx)

Изменить `handleAddSubscription` для вызова уведомления после успешной активации подписки:

**Изменения:**
1. После успешного вызова `supabase.rpc('activate_subscription', ...)` 
2. Вызвать `supabase.functions.invoke('send-subscription-notification', {...})`
3. Передать данные: userId, cupsCount, daysCount, type: 'activated'

### Шаг 3: Обновление Partner Scan QR Function

Добавить логику проверки остатка после списания и отправки уведомлений:

**Изменения в `partner-scan-qr/index.ts`:**
1. После успешного списания проверить новый остаток
2. Получить данные о подписке (сколько дней осталось)
3. Если остаток = 7, 5 или 3 — вызвать уведомление
4. Использовать фоновую задачу (`EdgeRuntime.waitUntil`) для асинхронной отправки

### Шаг 4: Обновление конфигурации

Добавить новую функцию в `supabase/config.toml`:
```toml
[functions.send-subscription-notification]
verify_jwt = false
```

## Технические детали

### Структура данных для уведомления об активации
```typescript
{
  type: 'activated',
  userId: string,
  cupsCount: number,    // количество кофе в подписке
  daysCount: number     // срок действия в днях
}
```

### Структура данных для уведомления о низком остатке
```typescript
{
  type: 'low_balance',
  userId: string,
  remaining: number,    // остаток кофе
  daysRemaining: number // дней до истечения
}
```

### Пороговые значения для уведомлений
- **7 кофе** — первое предупреждение
- **5 кофе** — второе предупреждение  
- **3 кофе** — финальное предупреждение

### Формат сообщений в Telegram

**Активация:**
```
☕ Подписка активирована

👤 Имя: {name}
📞 Телефон: {phone}
📦 30 кофе на 30 дней
🕐 {дата/время}
```

**Низкий остаток:**
```
⚠️ Низкий остаток по подписке

👤 Имя: {name}
📞 Телефон: {phone}
📦 Осталось 7 кофе на 25 дней
🕐 {дата/время}
```

## Синхронизация разделов

Уведомления будут отправляться единообразно из всех точек системы:
- ✅ **Админ панель** — при ручной активации подписки
- ✅ **Панель партнёра** — при сканировании QR и низком остатке
- ✅ **Приложение** — данные о подписке и остатке синхронизированы через существующие хуки
