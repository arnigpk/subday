
# План: Интеграция 2GIS API для расчёта расстояний

## Обзор решения

Подключим **2GIS Distance Matrix API** для расчёта реальных расстояний до кофеен по дорогам. Это даст более точные данные, чем расчёт "по прямой".

---

## Шаг 1: Получение API ключа 2GIS

Тебе нужно зарегистрироваться и получить ключ:

1. Перейди на https://dev.2gis.com/api
2. Нажми "Get Demo Key" для тестирования (бесплатно на 1 месяц, 1000 запросов)
3. Или оформи подписку для production-использования
4. Скопируй полученный API ключ

---

## Шаг 2: Изменения в базе данных

Добавим координаты для кофеен:

```text
Таблица: shops
Новые колонки:
  - latitude (DOUBLE PRECISION) — широта
  - longitude (DOUBLE PRECISION) — долгота
```

---

## Шаг 3: Edge Function для 2GIS API

Создадим серверную функцию для безопасного вызова 2GIS API:

```text
supabase/functions/calculate-distances/index.ts

Принимает:
  - user_lat, user_lng — координаты пользователя
  - shop_coords[] — массив координат кофеен

Возвращает:
  - distances[] — расстояния в метрах
  - durations[] — время в пути в секундах
```

API ключ 2GIS будет храниться как секрет, недоступный клиенту.

---

## Шаг 4: Хук геолокации пользователя

```text
src/hooks/useGeolocation.ts

- Запрашивает разрешение на геолокацию
- Использует watchPosition для обновления в реальном времени
- Возвращает { latitude, longitude, error, loading }
```

---

## Шаг 5: Хук для расстояний до кофеен

```text
src/hooks/useShopDistances.ts

- Получает координаты пользователя из useGeolocation
- Вызывает edge function calculate-distances
- Кэширует результаты (обновление каждые 30 сек)
- Возвращает Map<shopId, { distance, duration }>
```

## Fallback без геолокации

Если пользователь не дал доступ:
- Показывать "— м" вместо расстояния
- Сохранять сортировку по sort_order
- Предложить включить геолокацию
