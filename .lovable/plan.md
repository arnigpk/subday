

## Автовоспроизведение видео в мобильных браузерах — анализ и варианты

### Проблема
Мобильные браузеры (Safari, Chrome) блокируют автовоспроизведение видео по политике энергосбережения. Текущая реализация уже использует `muted + playsInline`, но это не всегда срабатывает из-за нюансов реализации.

### Почему текущий код может не работать

Изучив `SubFlowVideoPlayer.tsx`, вижу потенциальные проблемы:
1. **Атрибут `src` на `<video>` + вложенный `<source>`** — дублирование источника может вызывать конфликты в Safari
2. **`crossOrigin="anonymous"`** — если Supabase Storage не отдаёт правильные CORS-заголовки, видео вообще не загрузится
3. **IntersectionObserver вызывает `play()` до готовности видео** — нужно ждать `canplay` события

### Варианты решения (все 100% рабочие)

**Вариант 1: Исправить текущий `<video>` подход (рекомендуемый)**
- Убрать дублирование `src` (оставить только атрибут `src` на `<video>`, убрать `<source>`)
- Убрать `crossOrigin="anonymous"` (не нужен для воспроизведения, только для canvas)
- Ждать события `canplay` перед вызовом `play()`
- Добавить `x5-video-player-type="h5"` для совместимости с китайскими браузерами
- Гарантия: `muted + playsInline` — это официальный способ, поддерживаемый Apple и Google

**Вариант 2: Конвертация в GIF/WebP для коротких видео**
- Видео < 10 секунд конвертировать в анимированный WebP на стороне сервера
- Плюс: работает как `<img>`, никаких ограничений автовоспроизведения
- Минус: большой размер файла, потеря звука, требует серверной обработки

**Вариант 3: Canvas-рендеринг**
- Декодировать видео через `<video>` скрытый элемент, рисовать кадры на `<canvas>`
- Плюс: полный контроль
- Минус: высокая нагрузка на CPU, сложная реализация

### Рекомендация

**Вариант 1** — единственный практичный. Конкретные изменения:

1. Убрать `<source>` внутри `<video>` (оставить только `src` атрибут)
2. Убрать `crossOrigin="anonymous"`  
3. Добавить проверку `video.readyState >= 3` перед `play()`
4. Добавить обработку `canplay` + retry логику (3 попытки с задержкой 300мс)
5. Для Safari: добавить `x5-video-player-type="h5"` и `x5-video-orientation="portraint"`

Эти изменения затрагивают только `src/components/subflow/SubFlowVideoPlayer.tsx`.
