## Кабинет инвестора — План реализации

### Что будет сделано

1. **Новая роль `investor**` — добавляется в существующий enum `app_role` в БД. Суперадмин назначает роль через страницу управления пользователями.
2. **Таблица `investor_settings**` — хранит настройки для каждого инвестора:
  - `user_id` (uuid, уникальный)
  - `profit_percent` (numeric) — процент заработка, задаётся суперадмином
  - `note` (text) — комментарий
  - `created_at`, `updated_at`
  - RLS: инвестор видит только свою запись, суперадмин — все
3. **Управление в админке** — В AdminUsersPage при назначении роли `investor` появляется поле ввода процента. Суперадмин может менять процент в любой момент.
4. **Кабинет инвестора (`/investor`)** — отдельная страница с защитой по роли. Данные читаются в реальном времени из таблиц `subscription_transactions`, `user_subscriptions`, `investor_settings`.

### Технические шаги

**База данных (3 миграции):**

- Добавить `investor` в enum `app_role`
- Создать таблицу `investor_settings` с RLS
- Обновить функцию `has_role` (уже поддерживает новые роли автоматически)

**Фронтенд:**

- `src/pages/investor/InvestorDashboard.tsx` — главная страница кабинета
- `src/components/investor/InvestorProtectedRoute.tsx` — защита маршрута
- `src/hooks/useInvestorAuth.ts` — хук авторизации инвестора (на базе `useAdminAuth`)
- Обновить `useAdminAuth.ts` — добавить `isInvestor`
- Обновить `App.tsx` — добавить маршрут `/investor`
- Обновить `AdminUsersPage.tsx` — добавить роль `investor` + поле процента

**Логика расчётов (все на клиенте, read-only):**

- Активные подписки: `SELECT count(*) FROM user_subscriptions WHERE is_active = true`
- Транзакции за период: `SELECT * FROM subscription_transactions WHERE created_at BETWEEN ...`
- Заработок = сумма транзакций × процент инвестора / 100

Никаких edge functions не требуется — все данные доступны через существующие таблицы с соответствующими RLS-политиками.
