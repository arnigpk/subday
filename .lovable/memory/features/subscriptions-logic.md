# Memory: features/subscriptions-logic
Updated: now

Логика подписок и уведомлений:
1. В приложении ярлык выгоды отображается как «X ₸ за напиток» (кофе) или «X ₸ за ланч» (еда).
2. Система управления автоматическими уведомлениями в админке позволяет редактировать шаблоны для активации, включая специальный тип 'activated_special' для спецпредложений с текстом: «🎉 Подписка 🎁СПЕЦПРЕДЛОЖЕНИЕ🎁 успешно активирована! 🚀 Наслаждайтесь😌».
3. Триггеры уведомлений интегрированы в Edge Functions 'verify-payment' и 'paylink-webhook', обеспечивая отправку правильного типа сообщения (обычная активация или спецпредложение) пользователю в Telegram/Push.
4. Поддерживаются каналы Telegram, Push или оба одновременно. Шаблоны используют переменные {{subscription_name}}, {{count}}, {{unit}} и учитывают русскую грамматику (например, «2 дня», «2 напитка»).
5. Уведомления о промежуточных значениях (4, 3, 1) отключены.
6. Подписка автоматически деактивируется (is_active: false), если баланс напитков или ланчей становится равным нулю. Уведомление об исчерпании лимита удалено.
7. Все уведомления (low_balance, expiring_soon) из partner-scan-qr отправляются через edge function send-subscription-notification, которая читает настройки канала из auto_notification_templates и отправляет и в Telegram, и в Push при настройке "both".
