# Memory: features/subflow-social-system
Updated: now

Социальная механика #subFlow: включает систему подписок ('subflow_follows') и уведомлений ('subflow_notifications') через Push и Telegram (@subday_lgbot). 
1. Логика уведомлений разделена: в окне уведомлений приложения отображаются все события без исключений и задержек. Внешние алерты (Telegram/Push) для реакций и комментариев считают ГЛОБАЛЬНО по всем постам пользователя и отправляются при достижении milestones (3, 5, 10, 20, 50, 100). Подписчики и новые посты работают как раньше.
2. Настройка порогов, кулдауна и шаблонов сообщений (с поддержкой {{count}}, {{actor_name}}) осуществляется администратором в разделе «Автоуведомления». 
3. Техническая реализация: таблица 'subflow_reactions' защищена UNIQUE constraint на (user_id, post_id, reaction) и использует логику upsert для предотвращения дублей. 
4. UI включает колокольчик с Realtime-счетчиком и функционал подписки. В списке уведомлений используются стандартные иконки-заглушки вместо аватаров пользователей.
5. Навигация из уведомлений: строки кликабельны (ChevronRight, превью до 40 символов). При нажатии система прокручивает к посту и подсвечивает его 2 секунды.
6. Видеоплеер совместим с мобильными браузерами (Safari, Chrome): использует атрибуты muted, playsInline, webkit-playsinline, preload="auto". При блокировке автовоспроизведения показывает кнопку Play. Поддерживает tap-to-play/pause.
7. Загрузка медиа использует побайтовый XHR-прогресс (xhrUpload.ts) без лимита размера файла (ограничение только 30с для видео).
